## 一、产品定位与服务模型

- **产品名：** 上手吧（shangshouba）
- **定位：** 社区到家深度保洁服务平台，集成邻里拼团与裂变邀约
- **3 档服务：** 深度开荒（basic）/ 深度开荒 Plus / 深度开荒 Max，对应不同阶段需求
- **服务阶段：** 勘查 → 深处理 → 交付验收，逐阶段预约
- **排单核心：** 用户下单时选的是**第 2 次（深处理）档期**（整天），第 1 次和第 3 次逐阶段另约
- **早鸟赠送：** 提前 ≥2 个月下单，赠送第 1 次上门（勘查）

## 二、成团规则（已确认）

| 项目 | 决策 |

1. 被推荐人付款 → 返佣金额写入推荐人钱包 `pendingAmount`（待激活，不可提现）

## 五、定价公式（服务端计算）

```
毛价     = 面积 × ¥15/m²
新客立减 = min(面积 × ¥1/m², ¥200)
早鸟折扣 = 面积 × (¥2 if ≥60天 | ¥0 if <60天)
拼团折扣 = 面积 × ¥2/m²（成团后返钱包，不参与下单减价）
应付     = 毛价 - 新客 - 早鸟
价格底线 = ¥12/m²（折扣叠满不低于此价）
```

计算由 `cloudfunctions/unifiedOrder/index.js` 服务端执行。

## 六、订单状态机（haokang 主链）

```
待支付 → 待服务 → 待勘查 → 待深处理 → 深处理中 → 已交付
                  └→ 退款处理中 → 已退款
```

- 云函数写入：待支付（unifiedOrder）、待服务（paymentCallback）、退款处理中（applyRefund）
- 后台/运营触发：待勘查、深处理中、已交付、已退款

## 七、退款规则（已确认）

- 可退状态：仅 **"待服务"** 和 **"待勘查"**
- 时间窗口：服务日期前 **72 小时**（服务端强校验）
- 其他状态或超过窗口：不可退款

---

## 八、邀约规则（已确认）

- 邀请人必须**已下单**
- 邀请人与被邀请人必须在**同小区**（后端硬校验）
- 被邀请人扫码后享 **-¥2/m²** 优惠
- 双重校验：checkout 前端即时比对 + unifiedOrder 服务端终极校验

---

## 九、重复下单拦截（已确认）

- 规则：**同小区 + 同房号** 拦截，不允许重复下单
- 房号匹配：V1 精确匹配（`buildNormalizedRoomNo` 标准化后比对）

## 十四、UI 设计基线（已锁定）

设计标准详见 `DESIGN.md`，以下为核心约束摘要：

- **主色系：** `#F9E12A`（黄） + 黑 + 白 + 灰，严禁其他颜色
- **风格：** 扁平秩序、轻量层级、苹果式极简
- **表达：** 服务型表达，不做功能堆叠
- **分区：** 色调过渡，禁止实线分隔
- **按钮：** 文字 ≤ 6 字
- **卡片：** ≤ 3 层逻辑堆叠
- **弹窗：** 自定义组件，禁止 wx.showModal
- **Header：** 可用玻璃效果（backdrop-filter blur）
- **Footer：** 不透明纯白，固定底部
